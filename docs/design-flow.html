User Journey — From Landing to Study Session

How a barber student moves through the app: entry point, auth gate, mode selection, and session outcomes. Proposed — Phase 1 lander live, Phase 2 study modes pending Convex + Clerk integration.

Flowchart — User Journey

Entry + Discovery

Lander → Hub

  • fadejunkie.com opens to brand lander — no auth required
  • Single CTA: "Start Studying" gates the auth flow
  • Users who browse without clicking loop freely on the lander

Auth Gate

Clerk + Google OAuth

  • Unsigned users redirect to Clerk sign-in
  • Clerk delegates to Google OAuth provider
  • JWT returned → Convex session established
  • Already-signed users bypass — straight to Hub

Study Modes

Three Paths

  • Flashcards — flip / mark known, no timer
  • Solo Quiz — scored, topic-filtered
  • Group Mode — create or join live room via PIN
  • All modes write back to Convex progress tables
